## What this project is

Habitat Echo is a 2D top-down **Eclipse Phase 2nd-edition** fan game built in
**Godot 4.6** (a first game-dev project), developed with the **godot-ai** MCP
plugin. You play a Firewall agent on a habitat station, resleeving between morphs
to complete a mission while avoiding a hostile nanite swarm.

- **Project path:** `C:\GodotProjects\HabitatEcho\habitate-echo`
  (the git repo is the inner `habitate-echo` folder — note the missing 2nd "t").
- **GitHub remote:** `Curious-Conclusion/habitate-echo` (branch `main`).

### Current game systems (as built)
- Movement (WASD/arrows), interact (E), **Moxie burn (F)**.
- 3 morphs in `scripts/morph_manager.gd`:
  - **Octomorph** — spd 150 / hp 80 / ability WALL_CLING
  - **Synth** — spd 120 / hp 150 / ability VACUUM_SEAL
  - **Biomorph** — spd 170 / hp 100 / ability CYBERBRAIN
- Mission = 3 objectives in `scripts/mission_manager.gd`. **Intended path now:**
  `retrieve_stack` (Octomorph, via maintenance hatch) -> `scan_stack`
  (Biomorph, terminal) -> `vent_signal` (Synth, airlock).
- **Phase 1 (done):** a WALL_CLING **maintenance hatch** in the Server Alcove
  (560,60) hides the cortical stack — only the Octomorph can retrieve it. The old
  Med Bay sample locker is now a breadcrumb hint pointing to the hatch.
- **Phase 2 (built, see verification note):** Moxie is now a full stress system:
  - **Moxie burn (F, cost 30)** — morph-dependent: Octomorph = ink-and-scramble
    (2x speed + 3s invulnerability), Synth = EMP (disperses nanite swarms),
    Biomorph = medichine surge (restores HP). Logic in `player.gd`.
  - **Med Bay autodoc** interactable (60,300) — restores +40 Moxie, 2 doses.
  - **Derangement** below Moxie 35: control slows toward 0.5x as Moxie nears 0,
    and a red screen glitch (`scripts/glitch_overlay.gd`, `GlitchLayer`) flickers
    harder the lower it gets.
- **Phase 3 (done):** the Terminal no longer auto-completes `scan_stack`. It now
  opens a **hacking minigame** (`scripts/hacking_minigame.gd`, `HackingMinigame`
  CanvasLayer, built procedurally): a **4x4 node grid** reveals a connection
  route; retrace it with an **arrow-key cursor + [E] confirm**. Difficulty
  **escalates per failed hack** — route length starts at 4 and grows +1 each fail
  (cap 7) via a persistent `_fails` counter (the async hardens). A cyberbrain can
  press **F** to spend 20
  Moxie and auto-trace the next node. Success completes `scan_stack`; a wrong node
  = "async strikes back" (−15 Moxie + spawns a nanite swarm). Wired in `main.gd`
  via `_on_hack_succeeded` / `_on_hack_failed`.
- **Phase 4 (done):** the station now extends right into **Crew Quarters** (the
  map is ~984 wide; `Camera2D` zoom 1.5 + limits, `main.gd._process` trails the
  player). A **CrewNPC** there has branching dialogue (`show_lines_with_choices`):
  reassure / intimidate / ask. A `firewall_rep` int in `main.gd` tracks trust;
  reassuring earns the NPC's help and an **override code (77-ALEPH)**
  (`has_override`). At the Terminal, having the override offers a choice to
  **skip the hacking minigame** (instant `scan_stack`). Reputation = unlocks a
  shortcut.
- Nanite swarm chases the player, deals periodic damage + drains Moxie; now
  respects the player's invulnerability flag.
- Moxie still costs 15 to resleeve, 10 on death; Moxie 0 on death = EGO DEATH.
- Lore terminal with branching dialogue (`dialogue_box.gd`).
- Map = original 640x480 zones + the Crew Quarters extension (~984 wide total).
- **Art (in progress):** characters now use authored pixel-art PNG sprites under
  `res://art/` — `octomorph/`, `synth/`, `biomorph/` (4 dir x 2 walk frames each),
  `npc/crew.png`, `swarm/` (2 frames). Generated by stdlib Python scripts
  `art/gen_octomorph.py` and `art/gen_characters.py` (re-run to regenerate).
  `player.gd._build_morph_frames` loads per-morph art (falls back to the old
  procedural blobs for any morph without art); `nanite_swarm.gd` loads the swarm
  PNGs; `interactable.gd` gained an optional `sprite_path` (used by CrewNPC).
  Loaders use `Image.load` fallback so they work pre-import. **Interactable
  objects (hatch/autodoc/terminal/pod/locker/device/data pad) are still colored
  squares** — next art target.

## Workflow decisions
- **Verification:** live via the **godot-ai MCP**.
- **Navigating the running game:** input-sim timing is too coarse to land on small
  targets; temporarily lower the morph's `move_speed` in source (e.g. 150 -> 15),
  run with autosave=false, navigate using a target's PromptLabel `visible` flag as
  an in-range sensor, then revert. (See `godot-ai MCP quirks` memory.)
- **Scene edits:** Vector2/Color can't be written via `node_set_property` (values
  stringify in transit); hand-edit the `.tscn` and run with autosave=false.
